# HARUNGI — Редизайн UX: Десктоп + Мобиль

**Дата:** 2026-05-18  
**Скоп:** Сайт (Next.js на Vercel) — десктоп и мобильный браузер  
**Вне скопа:** Telegram Mini App (отдельный проект после завершения этого)

---

## Контекст

Текущий сайт "продаёт магазин" вместо того чтобы продавать духи. Главная перегружена блоками доверия (TrustSection, SocialProof, Guarantees), которые занимают экран вместо товаров. В каталоге фильтры спрятаны в sidebar с 8 секциями, цена и CTA видны только на hover (не работает на мобиле), нет нижней навигации на мобиле.

**Цель:** Минимум лишней информации, максимум товара. Всё ведёт в каталог.

**Стиль сохраняется:** тёплый люкс-минимализм, кремовая палитра, оливовый акцент, Georgia serif — только UX меняется.

**FeaturedPerfumes (расширяется до 4 товаров)**

**Строка доверия (новый компонент, заменяет 4 блока)**

## 2. Каталог

### Фильтры: sidebar → горизонтальные чипы

**Убирается:** левый sidebar (`w-52`) с 8 аккордеон-секциями (сезон, интенсивность, тип, формат, гендер, бренд, цена, новинки)

**Появляется:** горизонтальная строка чипов под поиском
- Первый ряд (всегда виден): `Все | Распивы | Оригиналы | — | Мужской | Женский | Унисекс | — | Древесный | Восточный | Свежий | Цветочный | + Ещё фильтры ▾`
- "Ещё фильтры" открывает bottom sheet (мобиль) или dropdown (десктоп) с расширенными опциями: бренд, цена, сезон, интенсивность
- Активные фильтры показываются как выделенные чипы в той же строке

**На мобиле:** чипы горизонтальный скролл без переноса + отдельная кнопка "⚙ Фильтры" справа от поиска

### Карточки товара: цена и CTA всегда видны

**Убирается:** hover-overlay (тёмная панель снизу с ценой и кнопкой)

**Появляется:** цена и кнопка видны всегда в нижней части карточки
- Десктоп: `[Бренд] [Название] [от X ₽] [Выбрать объём]`
- Мобиль: то же, кнопка `+ В корзину` (меньше текста)
- "Выбрать объём" / "+ В корзину" открывает bottom sheet с выбором объёма и кнопкой "Добавить в корзину" — без перехода на страницу товара и без OrderModal (не одиночный заказ, а добавление в корзину). Нужен новый компонент `QuickAddSheet`.

### Поиск

- Поисковая строка остаётся в каталоге, но становится более заметной (full-width, вверху)
- На мобиле: в шапке каталога, до чипов

### Компоненты для изменения
- `src/components/CatalogClient.tsx` — убрать sidebar, добавить chip-фильтры, обновить карточку
- `src/components/ProductCard.tsx` — убрать hover-overlay, добавить постоянный блок с ценой и CTA

## 3. Мобильная навигация: нижняя панель

### Новый компонент `BottomNav`

**Структура (3 пункта):**
```
[ ⌂ Главная ] [ ⊞ Каталог ] [ 🛒 Корзина (badge) ]
```

- Фиксированная, `position: fixed; bottom: 0`
- Фон `cream-50` (#fff), верхняя граница `cream-200`
- Safe area: `padding-bottom: env(safe-area-inset-bottom)`
- Badge на Корзине: зелёный кружок с числом, скрыт при 0 товарах
- Активный пункт: `ink-900` + подчёркивание или dot, неактивный: `ink-300`

### Изменения в layout

- `src/app/layout.tsx` — добавить `<BottomNav />` только на мобиле (`hidden sm:hidden md:block` инверсия — видна только на `< md`)
- Все страницы получают `pb-16` на мобиле для отступа от BottomNav
- `src/components/Header.tsx` — на мобиле: скрыть nav-links и кнопку "Заказать" (они дублируют BottomNav), оставить только логотип + иконку корзины. Корзина в хедере на мобиле скрывается — она уже есть в BottomNav.
